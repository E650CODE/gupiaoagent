## 一、项目目标
搭建一套完整 B/S 架构应用：用户通过网页端发起选股、趋势查询、策略回测请求；后端由多个专职 Agent 分工协作完成数据采集、因子计算、选股筛选、趋势预测、风险评估、回测分析；所有智能研判、策略解读、自然语言总结逻辑统一调用外部在线通用大模型 API，最终在网页可视化展示结果、K 线图表、风险提示。

### 2.2 通信方式
- **前后端**：HTTP/JSON，`/api/v1/*` 路径前缀，CORS 中间件放行本机端口；
- **主 Agent ↔ 子 Agent**：同进程 `async def handle(payload) -> AgentResult`；
- **Agent ↔ LLM**：统一 `llm_client.chat(agent_name, prompt, **opts)`；
- **缓存层**：Redis（热数据，TTL）+ SQLite（K线/基础信息/回测结果持久化）。

## 六、核心模块设计

### 6.1 主调度 Agent（MasterAgent）
- 接收三类请求：选股 / 趋势预测 / 回测；
- 编排子 Agent；超时控制（`asyncio.wait_for`，默认 60s）；
- `tenacity` 重试（3 次指数退避）；
- 异常兜底：单 Agent 失败时返回 `partial=true` + 已完成阶段结果；
- 每请求生成 `trace_id` 贯穿全链路日志。

| 业务 | 调度顺序 |
|------|---------|
| 选股 | DataAgent → FactorAgent → SelectorAgent(本地+LLM) → RiskAgent → 聚合 |
| 预测 | DataAgent → FactorAgent → PredictorAgent(LLM) → RiskAgent → 聚合 |
| 回测 | DataAgent → FactorAgent → SelectorAgent(历史) → BacktestEngine → LLM 总结 |

### 6.2 数据采集 Agent（DataAgent）
| 方法 | 数据源 | 缓存策略 |
|------|--------|---------|
| `get_all_stocks` | akshare 股票列表+行业 | SQLite 持久化 + Redis 1 天 |
| `get_kline(code, start, end, adjust='qfq')` | akshare/mootdx | SQLite 按日存储 + Redis 当日盘后失效 |
| `get_minutes(code)` | akshare 分时 | Redis 1 小时 |
| `get_tick(code, date)` | mootdx | SQLite 永久 |
| `get_money_flow(code)` | akshare 资金流 | Redis 1 小时 |
| `get_fundamental(code)` | akshare 指标 | SQLite 1 天刷新 |

预处理：剔除 ST/*ST/退市；剔除停牌（成交量=0）；剔除一字板（开收盘=最高=最低且触板）。

### 6.3 因子指标 Agent（FactorAgent）
基于 TA-Lib：MA(5/10/20/60)、MACD(12,26,9)、KDJ(9,3,3)、BOLL(20,2)、RSI(6/12/24)、量比(5)、换手率、乖离率(6)。

所有参数从 `strategies/presets.py` 读取，可被请求覆盖。极值用 1%/99% winsorize，空值前向填充。

### 6.4 多因子选股 Agent（SelectorAgent）
**内置策略**（可叠加）：
- `ma_bull`：MA5 > MA10 > MA20 > MA60；
- `macd_golden`：DIF 上穿 DEA 且零轴上方；
- `volume_surge`：量比 > 2 且收阳；
- `money_inflow`：近 5 日主力净流入 > 0；
- `low_valuation`：PE < 行业中位数且 PE > 0。

**LLM 解读**：将候选池因子快照 + 规则拼装 Prompt，输出选股逻辑、共性特征、板块热度。

**禁止未来函数**：所有指标只看截止 T 日数据。二次过滤 ST/高风险。

### 6.5 趋势预测 Agent（PredictorAgent）
- 提取近 60 日 K 线 + 技术指标 + 资金流摘要 → 结构化文本；
- 调用 LLM（JSON 模式）：
  ```json
  {"direction":"up|down|range","prob":0.0,"price_low":0,"price_high":0,
   "horizon_days":5,"reasoning":"..."}
  ```
- 置信度分级：高 ≥0.7、中 0.55–0.7、低 <0.55；
- 强制附带风险提示，禁止长期价值判断。

### 6.6 风控 Agent（RiskAgent）
**本地评级**：
| 等级 | 触发条件（任一） |
|------|----------------|
| 高 | 20日振幅>30% / 高位放量滞涨 / 连续3日跌幅>5% |
| 中 | 振幅15-30% / 资金净流出且MA走平 |
| 低 | 其他 |

LLM 生成风险点解读、止盈止损参考区间（基于波动率）、持仓提示。

### 6.7 回测 Agent + 引擎扩展点
**`engine_base.BacktestEngine` 抽象接口**：
```python
class BacktestEngine(ABC):
    @abstractmethod
    def run(self, strategy, kline_provider, params) -> BacktestResult
```
- 默认实现 `engine_simple.SimpleEngine`：每日按策略选股 → 等权买入 → 持有 N 日卖出 → 计算累计收益、年化、胜率、最大回撤、夏普、平均持仓收益；
- 预留 `engine_vectorbt.py` / `engine_backtrader.py` 等扩展实现，仅需注册至 `engine_registry`；
- LLM 总结：策略优缺点、参数优化建议。

### 6.8 LLM 统一客户端（核心变更）

**配置项（每 Agent 独立）**：
```json
{
  "selector": {
    "provider": "volcengine",
    "base_url": "https://ark.cn-beijing.volces.com/api/v3",
    "api_key": "ENC:gAAAAA...",          // Fernet 加密
    "model": "deepseek-v3-250324",
    "timeout": 60,
    "temperature": 0.3,
    "max_tokens": 2048
  },
  "predictor": { "provider": "deepseek", "base_url": "https://api.deepseek.com/v1", ... },
  "risk":      { ... },
  "backtest":  { ... }
}
```

**Provider 适配层 `llm_providers.py`**：
| Provider | base_url 模式 | list_models 端点 | 备注 |
|----------|-------------|-----------------|------|
| `volcengine` 火山方舟 | `https://ark.cn-beijing.volces.com/api/v3` | `GET /models`（带 Bearer Key） | OpenAI 兼容 |
| `deepseek` | `https://api.deepseek.com/v1` | `GET /models` | OpenAI 兼容 |
| `openai_compatible` | 自定义 | `GET /models` | 兜底通用适配 |

**`POST /api/v1/llm/models` 接口**：
- 入参：`{provider, base_url, api_key}`；
- 出参：`{models: ["deepseek-v3-250324","doubao-pro-32k", ...]}`；
- 用途：前端配置页填入 URL+Key 后，点击"获取模型列表"自动拉取，下拉选择，避免手填出错。

**`LLMClient.chat`**：
- 兼容 OpenAI ChatCompletion：`POST {base_url}/chat/completions`；
- tenacity 3 次指数重试；
- `json_mode=True` 时强制 JSON 返回并容错解析；
- 请求/响应全量落 `logs/llm_*.log`。

### 6.9 存储层（新增）

**SQLite（`backend/data/app.db`）**：
| 表 | 用途 |
|----|------|
| `stock_basic` | 全A股代码、名称、行业、上市日期 |
| `kline_daily` | 个股日线（code, date, OHLCV, amount, adjust） |
| `factor_snapshot` | 因子快照（按日存储，加速选股） |
| `backtest_record` | 回测任务结果与净值曲线 |
| `llm_audit` | LLM 请求审计（可选） |

**Redis**：
| Key 模式 | TTL | 说明 |
|----------|-----|------|
| `stocks:list` | 1d | 全市场列表 |

| `money:{code}` | 1h | 资金流 |
| `minutes:{code}` | 1h | 分时 |
| `select:{strategy_hash}` | 10min | 选股结果（含 AI 文案） |

1. **接口响应级**：所有 `/api/v1/*` 响应固定带 `disclaimer` 字段，由后端中间件强制注入；
2. **页面常驻级**：每个页面底部固定 `ComplianceFooter` 组件，文字红色突出；
3. **首次访问级**：用户首次进入弹出合规确认弹窗（"我已知悉风险"），结果写入 `localStorage`，关闭浏览器后下次仍生效一段时间（30 天），到期再次弹出。

## 九、非功能性需求
- **日志**：loguru 控制台+文件，按日切割；每请求 trace_id 贯穿全链路；
- **配置**：`.env` + `llm_config.json`（Fernet 加密）；所有密钥、超时、参数集中管理；
- **跨域**：FastAPI CORSMiddleware 放行 `http://localhost:5173`、`http://127.0.0.1:5173`；
- **异常捕获**：全局 ExceptionHandler 捕获 `HTTPException` / `RequestException` / 未知异常，统一返回 `{code,msg,trace_id,disclaimer}`，绝不裸抛 500；
- **请求重试（统一规范）**：
  - **数据源接口**（akshare/mootdx/资金流/分时）：`@retry(stop=stop_after_attempt(3), wait=wait_exponential(min=1,max=8), retry=retry_if_exception_type((ReadTimeout,ConnectionError)))`；
  - **LLM 调用**：同上策略，3 次指数退避；
  - **子 Agent 调用**：MasterAgent 内对每个子 Agent `tenacity` 1 次额外重试（避免与底层重试叠加放大）；
- **本地缓存（双层 + 自动降级）**：
  - L1 Redis 热缓存（TTL 见 §6.9）；
  - L2 SQLite 持久化（K线/股票列表/因子快照/回测结果）；
  - **Redis 不可用时自动降级到进程内 `cachetools.TTLCache`，应用不阻塞启动**，日志 WARN 提示；
- **测试矩阵（强制最小集）**：
  | 模块 | 用例数 | 覆盖 |
  |------|-------|------|
  | DataAgent | ≥2 | 正常拉取 + 接口超时降级 |
  | FactorAgent | ≥1 | 固定K线断言指标数值 |
  | SelectorAgent | ≥1 | 本地筛选（mock LLM） |
  | PredictorAgent | ≥1 | JSON 解析容错 |
  | RiskAgent | ≥1 | 评级规则边界 |
  | BacktestAgent | ≥1 | 自研引擎收益/回撤断言 |
  | LLMClient | ≥2 | 重试机制 + 模型列表拉取 |
  | API 集成 | ≥1 | `/health` + 1 个业务路由 |
- **性能**：首次选股全市场 ≈ 30-60s（akshare 瓶颈），Redis 命中后 ≈ 2-5s；
- **安全**：API Key Fernet 落盘，前端 GET 时返回 `sk-****abcd` 掩码；
- **合规**：见 §8 三重保险。
